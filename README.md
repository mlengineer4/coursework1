# FocusTomato — визуальный Pomodoro-таймер

## Описание проекта

FocusTomato — это учебный веб-проект, разработанный для курсовой работы по теме **«Разработка веб-интерфейса сайта визуального таймера Pomodoro»**.

Сайт представляет собой визуальный таймер для организации работы по методу Pomodoro. Пользователь может запускать таймер, ставить его на паузу, сбрасывать, переключать режим работы и перерыва, просматривать статистику и изменять длительность интервалов.

Проект выполнен с использованием базовых веб-технологий:

- HTML;
- CSS;
- JavaScript;
- localStorage.

Код проекта разделён на отдельные файлы, что делает структуру сайта понятной и удобной для объяснения.

---

FocusTomato/

## Этапы разработки проекта

### 1. Анализ темы курсовой работы

На первом этапе была определена основная цель проекта — создать веб-интерфейс визуального таймера Pomodoro. Было решено, что сайт должен быть не просто таймером, а полноценным интерфейсом с несколькими страницами, настройками и статистикой.

Были выделены основные функции сайта:

- запуск таймера;
- пауза таймера;
- сброс таймера;
- переключение между режимом работы и перерыва;
- визуальное отображение прогресса;
- сохранение статистики;
- изменение настроек;
- переход между страницами сайта.

---

### 2. Проектирование структуры сайта

На втором этапе была определена структура проекта. Вместо одной страницы было решено создать несколько HTML-страниц:

- главная страница с таймером;
- страница статистики;
- страница с описанием метода Pomodoro;
- страница настроек.

Такой подход позволил сделать сайт более похожим на полноценный веб-интерфейс, а не на один отдельный таймер.

---

### 3. Создание HTML-разметки

На третьем этапе была создана HTML-структура страниц. В HTML были добавлены основные элементы интерфейса:

- шапка сайта;
- логотип;
- навигационное меню;
- карточка таймера;
- кнопки управления;
- блок визуального круга;
- блок статистики;
- форма настроек;
- информационные карточки.

Каждая страница получила собственный HTML-файл, но все страницы используют общий CSS-файл и общий JavaScript-файл.

---

### 4. Разработка визуального оформления CSS

На четвёртом этапе был создан файл `style.css`, который отвечает за внешний вид сайта.

В CSS были реализованы:

- тёмный градиентный фон;
- полупрозрачные карточки;
- скругления элементов;
- стилизация кнопок;
- hover-эффекты;
- визуальный круг таймера;
- адаптивная сетка;
- анимация появления страниц;
- оформление навигационного меню.

Целью оформления было сделать интерфейс современным, понятным и визуально приятным.

---

### 5. Реализация логики таймера на JavaScript

На пятом этапе была разработана основная логика таймера в файле `script.js`.

В JavaScript были созданы переменные для хранения:

- длительности рабочего режима;
- длительности перерыва;
- текущего режима;
- общего количества секунд;
- оставшегося времени;
- состояния запущенного таймера.

Таймер работает через `setInterval`, который каждую секунду уменьшает значение `remainingSeconds`. После каждого изменения обновляется текст времени и визуальный круг прогресса.

---

### 6. Реализация кнопок управления

На шестом этапе были подключены кнопки управления таймером:

- `Старт`;
- `Пауза`;
- `Сброс`;
- `Работа`;
- `Перерыв`.

Для каждой кнопки была назначена отдельная JavaScript-функция:

| Кнопка | Функция |
|---|---|
| Старт | `startTimer()` |
| Пауза | `pauseTimer()` |
| Сброс | `resetTimer()` |
| Работа | `setWorkMode()` |
| Перерыв | `setBreakMode()` |

Кнопка «Старт» запускает обратный отсчёт, «Пауза» останавливает его без сброса, а «Сброс» возвращает время к начальному значению текущего режима.

---

### 7. Реализация визуального круга прогресса

На седьмом этапе был реализован визуальный индикатор таймера.

JavaScript рассчитывает:

- сколько секунд прошло;
- какой процент времени уже завершён;
- сколько градусов круга нужно закрасить.

После этого CSS-свойство `background` у круга изменяется с помощью `conic-gradient`.

Благодаря этому пользователь видит прогресс не только в виде цифр, но и визуально.

---

### 8. Реализация статистики

На восьмом этапе была добавлена статистика продуктивности.

Сайт сохраняет:

- количество завершённых Pomodoro;
- общее количество минут фокуса;
- уровень продуктивности.

Статистика обновляется после завершения рабочего интервала. Если таймер заканчивается в режиме работы, количество Pomodoro увеличивается на 1, а к минутам фокуса добавляется длительность рабочего интервала.

---

### 9. Использование localStorage

На девятом этапе было добавлено сохранение данных в браузере с помощью `localStorage`.

В `localStorage` сохраняются следующие данные:

| Ключ | Значение |
|---|---|
| `workMinutes` | длительность работы |
| `breakMinutes` | длительность перерыва |
| `pomodoroCount` | количество завершённых Pomodoro |
| `focusMinutes` | количество минут фокуса |

Использование `localStorage` позволяет сохранять статистику и настройки даже после обновления страницы или перехода между страницами сайта.

---

### 10. Реализация страницы настроек

На десятом этапе была создана страница настроек. На ней пользователь может изменить:

- количество минут работы;
- количество минут перерыва.

При сохранении JavaScript проверяет, чтобы введённые значения были больше нуля. Если данные корректны, они сохраняются в `localStorage`.

---

### 11. Реализация многостраничной навигации

На одиннадцатом этапе была реализована навигация между страницами сайта.

Переходы выполнены через обычные HTML-ссылки:

```html
<a href="index.html">Таймер</a>
<a href="stats.html">Статистика</a>
<a href="about.html">О методе</a>
<a href="settings.html">Настройки</a>
```

Такой способ простой, понятный и подходит для учебного проекта.

---

### 12. Проверка работы сайта

На последнем этапе была выполнена проверка основных функций:

- запуск таймера;
- пауза таймера;
- сброс таймера;
- переключение режимов;
- завершение рабочего интервала;
- сохранение статистики;
- очистка статистики;
- изменение настроек;
- переход между страницами.

После проверки сайт был подготовлен к использованию в курсовой работе.

## Использованные технологии

### HTML

HTML использовался для создания структуры страниц сайта: кнопок, заголовков, карточек, меню, форм и блоков с информацией.

### CSS

CSS использовался для оформления сайта: цветов, фона, карточек, кнопок, отступов, анимаций и адаптивности.

### JavaScript

JavaScript использовался для реализации интерактивности: запуска таймера, паузы, сброса, переключения режимов, обновления визуального круга, сохранения статистики и настроек.

### localStorage

`localStorage` использовался для сохранения пользовательских данных в браузере.

## Основной сценарий использования

1. Пользователь открывает главную страницу сайта.
2. Пользователь выбирает режим работы или перерыва.
3. Пользователь нажимает кнопку «Старт».
4. Таймер начинает обратный отсчёт.
5. Время и визуальный круг обновляются каждую секунду.
6. После завершения рабочего интервала статистика сохраняется.
7. Сайт переключается в режим перерыва.
8. Пользователь может посмотреть статистику или изменить настройки.
